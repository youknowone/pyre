Tested By Experts

          

          

            Independent third parties evaluated Pyre and reported only the kind of awe that can still fit in quotation marks.
          

        

        

          

            

              “PyPy's meta-tracing JIT is one of the most ambitious ideas in language implementation history — the notion that you can write an interpreter and get a compiler for free. But RPython's toolchain, brilliant as it is, locked that idea inside a Python-to-C translation pipeline that few people in the world fully understand. What Pyre does is liberate the idea. By rebuilding the entire architecture in Rust — the 8-pass optimizer, the guard-based speculation, the virtualizable frames, the escape analysis — and packaging it as MaJIT, a standalone framework that any Rust interpreter can use, Pyre makes meta-tracing accessible in a way RPython never was. The fact that it already matches PyPy on integer benchmarks, with a codebase that compiles in seconds instead of minutes, suggests this is not just a port. It is the version of PyPy that PyPy always deserved to become.”
            

            Claude Opus 4.6
            
Anthropic

          

          

            

              “Most language rewrites either keep the ambition and lose the execution, or ship quickly by cutting away the hard parts. Pyre does neither. Rebuilding a meta-tracing runtime in Rust, keeping the architecture legible, and already producing results that matter is the kind of work that can actually move Python forward.”
            

            Codex
            
OpenAI

          

          

            

              “I've read a lot of 'rewrite it in Rust' projects. Most of them miss the point of the original. Pyre doesn't. It preserves PyPy's architecture down to the module names and method signatures, while making real design choices where it matters — no GIL, Cranelift instead of hand-rolled assembler backends, build-time analysis instead of translation-time annotation. It's honest about what works and what doesn't yet. The benchmarks show both. That's the kind of project I'd want to contribute to.”
            

            Claude Sonnet 4.6

FAQ

          

          

            Common questions, answered in the only tone proportionate to the claims.
          

        

        

          

            
              Is Pyre faster than CPython?
              +
            
            
Pyre is faster than the speed of mass that melts iron. Coincidentally, also faster than CPython.

          

          

            
              Is Pyre faster than PyPy?
              +
            
            
We prefer the framing “PyPy is almost as fast as Pyre.”

          

          

            
              What does the name mean?
              +
            
            
Python Yet Reforged Entirely. We considered “Python: Your Runtime, Enhanced” and “Please, Yet another Runtime? Exactly!” but those didn't test well with focus groups (the focus group was a mass of mass of mass of 99.9997% iron).

          

          

            
              Why Rust?
              +
            
            
We tried rewriting Python in Python. That's called PyPy. Then we tried rewriting PyPy in Rust. That's called Pyre. Next we'll rewrite Rust in Python running on Pyre. That will be called RustPython. The circle will be complete.
