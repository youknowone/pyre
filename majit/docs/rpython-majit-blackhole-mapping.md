### 1.3 bhimpl_* instruction implementations (blackhole.py:454-1607)

These are the ~150 instruction handlers. In majit, they correspond to
opcodes in `dispatch_one()` (blackhole.rs:1141). The jitcode-based
blackhole already handles core opcodes (BC_LOAD_CONST, BC_MOVE,
BC_PUSH, BC_POP, BC_JUMP, BC_COND, BC_CALL, BC_RETURN, etc.).

**Gap**: Application-specific opcodes (Aheui storage ops, I/O) are
handled via `fn_ptrs` in JitCode, not as bhimpl methods. This is an
acceptable structural difference since pyre generates the JitCode.

## 6. Structural Differences (Acceptable)

| Aspect | RPython | majit | Why acceptable |
|--------|---------|-------|----------------|
| Dispatch mechanism | Generated `dispatch_loop` from insn table | Hand-written `dispatch_one` match | Rust has no eval-time metaprogramming; match is equivalent |
| bhimpl_* methods | ~150 methods with @arguments decorator | JitCode.fn_ptrs function pointers | pyre generates JitCode with correct fn_ptrs |
| Builder pattern | Class-based factory with pool | Struct-based builder with Vec pool | Same semantics |
| VirtualInfo hierarchy | 14 classes with allocate() methods | Single enum with 14 variants | Rust enum dispatch is equivalent |
| Constants in registers | Copy into register array tail | setposition allocates with constants | Same layout, different init path |
| resumecodereader | Custom numb_next_item() with varint | i64 Vec with direct indexing | Encoding differs but semantics same |
